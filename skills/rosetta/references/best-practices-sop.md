分析原因：
1. 查询未使用索引，全表扫描

执行计划：
- type: ALL（全表扫描）

- Extra: Using where

问题确认：
✗ 未使用索引
✗ 全表扫描 8900 万行数据
✗ JSON 字段查询效率低

优化建议：

1. 添加虚拟列和索引（推荐）

```sql
ALTER TABLE large_table
```

```json

  "timeout": 600
}
```
